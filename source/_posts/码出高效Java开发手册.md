# oop理念

oop 全称 Object-Oriented Programming ，面向对象编程。这的确是个跨时代的想法。因为一开始学的是C语言，所以对面向过程的编程有一定的了解，面向过程的确就像一条流水线，按书里说的一样，面条代码，流程互相穿插，理还乱。面向对象的编程为什么算是很伟大的想法，就因为它能很有效的解决这样面条代码的问题， 类的出现将程序将面向对象的条状互相穿插的状态改成了算是横向的穿插，横向的发展的确能解决很多流程很难解决的问题。

# 关于继承

继承一开始是为了更好的解决程序代码重复的问题，为了更好的开发。它的使用也很简单，只用一个extends就能获得父类的方法，不过就因为太简单了，所以就会出现了滥用的问题，就像书里说的，滥用继承出现了方法污染和方法爆炸的危害。所以对于开发来说还是对继承持尽量不用的态度。

> 1. 方法污染。子类继承了父类具备的行为，但无法执行父类的的方法。
> 2. 方法爆炸。爆炸是指继承树的扩大，很多个子类就会产生很多个相同的方法，形成方法爆炸。

# 接口与抽象类

类是对具体事物的抽象，接口与抽象类算是更高层次的抽象。

抽象类算是一种模板式设计，而接口是一种契约式的设计。
